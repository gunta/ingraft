# Ingraft — Brand Reference

Internal brand documentation. Use this as the source of truth when writing copy, slides, docs, or anything else that surfaces the Ingraft name.

## Quick facts

- **Name:** ingraft (lowercase as identifier; "Ingraft" at sentence start in prose)
- **Pronunciation:** in-GRAFT — /ɪnˈɡræft/
- **What it is:** a CLI + agent skill that vendors upstream source from GitHub into your repo so coding agents have full, local context
- **Origin:** born from the Effect article ["The One Weird Git Trick That Makes Coding Agents More Effect-ive"](https://effect.website/blog/the-one-weird-git-trick-that-makes-coding-agents-more-effect-ive/)
- **npm:** [ingraft](https://www.npmjs.com/package/ingraft), [@ingraft/skill](https://www.npmjs.com/package/@ingraft/skill)
- **Domain:** [ingraft.dev](https://ingraft.dev)

## Audience

Ingraft is for:

1. **Developers** building applications who use coding agents (Claude Code, Codex, Cursor, etc.) and want those agents to have proper local context for every dependency.
2. **Agent runtimes** themselves — agents invoke `bunx ingraft` to vendor what they need.
3. **The Effect / TypeScript ecosystem** as the launch community (the article that spawned the tool came from there).
